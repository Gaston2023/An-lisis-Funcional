# 🧩 Sistema de Gestión de Tareas – Análisis Funcional

## 📌 Descripción

Este proyecto consiste en el análisis, modelado y diseño funcional de un sistema web para la gestión de tareas dentro de una organización.

El sistema permite administrar usuarios y tareas, asignar responsabilidades, realizar seguimiento del estado de las tareas y mantener un historial de cambios.

Este trabajo fue realizado como parte de la materia **Ingeniería de Software**, aplicando metodologías de análisis orientadas a objetos (UML y OOWS).

---

## 🎯 Objetivo del Proyecto

El objetivo principal es definir de manera clara y estructurada los requerimientos del sistema antes de su desarrollo, utilizando herramientas de análisis funcional y modelado.

---

## 🧠 Alcance Funcional

### 👤 Gestión de Usuarios

* Alta de usuarios
* Autenticación
* Consulta y modificación
* Suspensión de usuarios

### 🗂️ Gestión de Tareas

* Creación de tareas
* Asignación a usuarios
* Modificación y seguimiento de estado
* Consulta de tareas por prioridad

### 📊 Gestión de Historial

* Registro de cambios en tareas
* Consulta del historial de operaciones

---

## 📋 Requerimientos

### ✔️ Funcionales

Definen las acciones que el sistema debe permitir:

* Gestión completa de usuarios y tareas
* Registro de operaciones
* Notificaciones ante cambios relevantes

### ⚙️ No Funcionales

Definen características de calidad del sistema:

* Seguridad (autenticación, control de sesiones)
* Escalabilidad y rendimiento
* Disponibilidad 24/7
* Accesibilidad web
* Usabilidad

---

## 🔒 Reglas de Negocio

* No se pueden eliminar tareas asignadas
* Los usuarios no pueden crear tareas
* No se puede suspender un usuario con tareas pendientes
* Las modificaciones de tareas deben registrarse en el historial
